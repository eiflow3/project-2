# Q&A: Testing Windows Builds on macOS (Without a Physical Windows Device)

If you are developing on a Mac and need to verify the build `.zip` or `.exe` of your desktop application generated by your GitHub Actions CI/CD pipeline, you don't need a physical Windows PC. There are two primary workflows you can use:

1. **Option A (Local)**: Translate or virtualize Windows directly on your macOS machine.
2. **Option B (CI/CD Remote)**: Launch an interactive graphical Remote Desktop session inside the GitHub Actions runner itself.

---

## 💻 Option A: Direct Local Emulation / Virtualization on macOS (Highly Recommended)

These options run locally on your Mac, meaning they do not consume GitHub Actions runner minutes and do not require external network configurations.

---

### 🏆 Choice 1: UTM (Free & Open Source Virtualization)
UTM is the gold standard for virtualization on Apple Silicon Macs. It allows you to run a full instance of Windows 11 ARM64 with near-native performance. Windows 11 ARM64 automatically and invisibly translates standard x64 Windows applications (like your compiled app) using its high-performance emulator.

#### Step-by-Step Installation Guide:

1. **Download and Install UTM**:
   - Go to [getutm.app](https://getutm.app/) and download the free installer (or purchase it from the Mac App Store to support the developers). Install it into your `/Applications` folder.

2. **Download Windows 11 ARM64 ISO**:
   - Open UTM, click the **+ (Create a New Virtual Machine)** button.
   - Select **Virtualize** -> **Windows**.
   - Tick the option **Install Windows 10 or newer** and click **Download Windows Installer ISO**.
   - This will open UTM's integrated downloader (or CrystalFetch), which pulls the official, clean ARM64 installer image directly from Microsoft's servers. Save this ISO file on your Mac.

3. **Configure the Virtual Machine**:
   - Back in UTM, browse and select the downloaded ISO file.
   - **Hardware Settings**:
     - **Memory (RAM)**: Allocate at least `4096 MB` (4GB). If your Mac has 16GB or more of RAM, allocate `8192 MB` (8GB) for a much smoother experience.
     - **CPU Cores**: Set this to `4` cores (perfect sweet spot for Apple Silicon).
   - **Storage**: Allocate at least `64 GB` (Windows installs require ~20GB, leaving plenty of room for developer runtimes and database testing).
   - **Shared Directory**: Choose a folder on your Mac (e.g., your desktop or workspace folder) to be visible inside Windows for instant file sharing.
   - Click **Save**.

4. **Boot and Bypass the Internet Connection Block (CRITICAL STEP)**:
   - Start the VM by clicking the large **Play** button.
   - Follow the regular Windows installation prompts. When the VM reboots into the initial setup wizard (OOBE), you will reach a screen saying: **"Let's connect you to a network"** with the "Next" button disabled.
   - **How to bypass this**:
     1. Press `Shift + F10` on your keyboard (on Mac keyboards, you may need to press `Fn + Shift + F10`).
     2. A black Command Prompt window will appear.
     3. Type exactly `OOBE\BYPASSNRO` and hit **Enter**.
     4. The virtual machine will reboot automatically.
     5. Now, on the same connection screen, you will see a new option: **"I don't have internet"**. Click it, and then click **"Continue with limited setup"**.
     6. Complete the setup to create a standard offline local account.

5. **Install SPICE Guest Tools (For Network, Clipboard, and Display Resolution)**:
   - Once logged onto the Windows desktop, you'll notice there is no internet connection and the screen resolution doesn't resize dynamically. We must install the guest drivers.
   - At the top of the UTM VM window, click the **CD/DVD icon** (drive controller) and select **Mount SPICE Guest Tools**.
   - Open **File Explorer** inside Windows, navigate to **This PC**, and double-click the CD/DVD drive (usually `D:`).
   - Run the installer executable (e.g., `spice-guest-tools-xxx.exe`) and accept all defaults/prompts.
   - Reboot the virtual machine.
   - **Result**: You now have full internet access, seamless clipboard copy-pasting between macOS and Windows, and the Windows display resolution will dynamically scale when you resize the UTM window!

### ⚡ Choice 2: Whisky (Free & Ultra-Lightweight Translation Layer)
Whisky uses Apple's **Game Porting Toolkit** and **Wine** to run Windows executables (`.exe`) directly on macOS without installing a full virtual machine. It is perfect if you want instant, zero-overhead testing of your compiled zip files.

#### Step-by-Step Installation Guide:

1. **Install Whisky**:
   - Run the following terminal command on your Mac to install it via Homebrew:
     ```bash
     brew install --cask whisky
     ```
   - Alternatively, download the DMG directly from [getwhisky.app](https://getwhisky.app/).

2. **Initialize Whisky**:
   - Launch the Whisky app from your Applications folder.
   - Follow the prompt to download and configure Apple's native **GPTK (Game Porting Toolkit)** and Wine dependencies (Whisky does this automatically with a single click).

3. **Create a Bottle**:
   - In Whisky, click **Create Bottle** (a "Bottle" is an isolated Windows environment prefix).
   - Give it a name (e.g., `OrderManagerTesting`), select **Windows 10** or **Windows 11** as the version, and click **Create**.

4. **Run Your Application**:
   - In your newly created bottle, click **Run...** at the bottom right.
   - Select your compiled `.exe` file (or extract your build `.zip` somewhere on your Mac and select the executable).
   - Whisky will boot and execute the app instantly right on your macOS Desktop as if it were a native Mac app!

## 🚀 Option B: Interactive Remote Desktop (RDP) on GitHub Actions Runner

If you prefer to inspect how the application behaves on a clean, native Windows 10/11 environment inside the GitHub Actions machine, you can run an **interactive GUI session** directly on the runner and connect visually from your Mac.

### How it Works
1. Create a manually triggered GitHub Actions workflow (`workflow_dispatch`).
2. The workflow compiles your Windows build, enables Remote Desktop (RDP), starts a secure tunnel using **ngrok**, and displays a public connection string.
3. You connect directly from your Mac using the **Windows App** (formerly *Microsoft Remote Desktop*, available on the Mac App Store).

### Step-by-Step Implementation

#### Step 1: Create an ngrok Account & Get a Token
1. Sign up for a free account at [ngrok.com](https://ngrok.com).
2. Go to your ngrok dashboard and copy your **Authtoken**.
3. In your GitHub repository, navigate to **Settings > Secrets and variables > Actions**.
4. Create a new repository secret named `NGROK_AUTH_TOKEN` and paste your authtoken.

#### Step 2: Create the Workflow File
Create a new file in your project under `.github/workflows/interactive-rdp-debug.yml`.

Here is a fully documented, transparent workflow utilizing native PowerShell scripts to avoid risky black-box third-party actions:

```yaml
# Workflow to build the Windows app and provide an interactive Remote Desktop session for testing.
name: "Interactive Windows Build Testing"

on:
  # Trigger manually from the GitHub Actions tab to avoid wasting runner minutes.
  workflow_dispatch:

jobs:
  build-and-test:
    runs-on: windows-latest
    steps:
      # Step 1: Pull repository code down to the Windows runner.
      - name: Checkout Code
        uses: actions/checkout@v4

      # Step 2: Set up the build environment.
      # (This is where your typical build setup goes. For Flutter, we configure Java, Flutter, and run pub get.)
      - name: Set up Flutter
        uses: subosito/flutter-action@v2
        with:
          channel: 'stable'
          cache: true

      # Step 3: Run the compile script to generate the Windows release.
      - name: Build Windows Application
        run: |
          flutter pub get
          flutter build windows --release

      # Step 4: Configure Remote Desktop (RDP) & start the secure ngrok tunnel.
      - name: Enable Interactive Remote Desktop
        shell: powershell
        run: |
          # 1. Enable RDP in the Windows Registry
          Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' -Name "fDenyTSConnections" -Value 0
          
          # 2. Allow Remote Desktop traffic through the local Windows Firewall
          Enable-NetFirewallRule -DisplayGroup "Remote Desktop"
          
          # 3. Create a secure password for the GitHub Runner administrator account (runneradmin)
          $Password = ConvertTo-SecureString "TestWindowsPass123!" -AsPlainText -Force
          Set-LocalUser -Name "runneradmin" -Password $Password
          
          # 4. Download and extract the native ngrok executable for Windows
          Invoke-WebRequest -Uri "https://bin.equinox.io/c/4VmDzA7iaHb/ngrok-stable-windows-amd64.zip" -OutFile "ngrok.zip"
          Expand-Archive -Path "ngrok.zip" -DestinationPath "."
          
          # 5. Authenticate ngrok using the secret token stored in GitHub
          .\ngrok.exe config add-authtoken "${{ secrets.NGROK_AUTH_TOKEN }}"
          
          # 6. Spawn the ngrok tunnel forwarding RDP (port 3389) in the background
          Start-Process -FilePath ".\ngrok.exe" -ArgumentList "tcp 3389" -NoNewWindow
          
          # 7. Give the ngrok process a few seconds to boot and establish connection
          Start-Sleep -Seconds 5
          
          # 8. Query ngrok's local API to fetch the external public URL
          $Response = Invoke-RestMethod -Uri "http://localhost:4040/api/tunnels"
          $PublicUrl = $Response.tunnels.public_url
          
          # Remove the 'tcp://' prefix from the returned url for easy copy-pasting
          $ConnectUrl = $PublicUrl -replace 'tcp://', ''
          
          # Print the connection credentials on the screen
          Write-Host "==================================================================" -ForegroundColor Green
          Write-Host "*** NATIVE RDP SERVER IS READY! ***" -ForegroundColor Green
          Write-Host "=================================================================="
          Write-Host "1. Connect using Windows App (formerly Remote Desktop) on your Mac to:"
          Write-Host "   --> $ConnectUrl" -ForegroundColor Yellow
          Write-Host "2. Login Credentials:"
          Write-Host "   Username: runneradmin"
          Write-Host "   Password: TestWindowsPass123!"
          Write-Host "=================================================================="
          Write-Host "Press the Cancel Workflow button in GitHub when you are finished."
          Write-Host "=================================================================="
          
          # 9. Infinite loop to keep the VM alive (runner terminates automatically after 6 hours maximum)
          while ($true) { Start-Sleep -Seconds 30 }
```

#### Step 3: Connect from your Mac
1. Install the free **Windows App** (formerly *Microsoft Remote Desktop*) from the Mac App Store.
2. Run your manual workflow on GitHub.
3. Open the workflow console logs, and find the step **Enable Interactive Remote Desktop**.
4. Copy the connection host and port (e.g., `0.tcp.ngrok.io:12345`).
5. Open the **Windows App**, add a new PC with that PC Name, and use:
   * **Username**: `runneradmin`
   * **Password**: `TestWindowsPass123!`
6. Click connect! You are now physically logging into the Windows VM in the cloud.
7. Go to `D:\a\<your-repo>\<your-repo>\build\windows\x64\runner\Release\` (or search the file system) to locate, open, and run the compiled Windows binary files directly on the runner with full graphics and execution!
